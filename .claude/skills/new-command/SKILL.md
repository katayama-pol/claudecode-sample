# 新しいサブコマンドを追加する手順

task-cli に新しいサブコマンド（例: `delete`, `export` など）を追加するときは、
必ずこの手順どおりに進めること。引数で追加したいコマンド名を受け取る。

## 手順

1. **既存実装の確認**: [src/index.ts](../../../src/index.ts) の `add` / `list` / `done`
   の書き方を読み、同じスタイルに合わせる。
2. **関数の追加**: 新しいコマンド用の `async function` を `index.ts` に追加する。
   - データの読み書きは必ず [src/store.ts](../../../src/store.ts) 経由にする。
   - 必要なら store.ts にヘルパー関数を足す。
3. **ルーティング**: `main()` の `switch` に `case` を追加する。
4. **ヘルプ更新**: `help()` の使い方一覧に1行追加する。
5. **テスト**: ロジックがあれば `src/store.test.ts` にテストを追加する。
6. **動作確認**: 実際に `node src/index.ts <新コマンド>` を実行して動くことを確認する。
7. **報告**: 追加したコマンドの使い方を日本語で1〜2行にまとめて報告する。

## 注意

- ユーザー向けメッセージはすべて日本語（プロジェクト規約に従う）。
- この手順を飛ばして直接実装を始めないこと。まず手順1から順に進める。
