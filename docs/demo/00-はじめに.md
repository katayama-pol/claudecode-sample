# 00. はじめに — このデモの歩き方

## ゴール

Claude Code の7つの機能を、**実際に効果を目で見て**理解することです。
「説明を読む」のではなく「打って・観察する」ことを重視しています。

## 進め方の基本

各ガイドはこの3点セットで書かれています。

1. **どんな機能か**（1〜2行）
2. **打つプロンプト**（Claude Code にそのまま入力する文）
3. **何が起きれば成功か**（観察ポイント）

番号順（01→07）に進めると、「自動で効くもの」→「明示的に呼ぶもの」の順で理解が積み上がります。

## 観察のコツ：「ON/OFF の差」を見る

機能の効果は、**無い状態と比べる**と一番よく分かります。
各ガイドには「比較実験」を用意しているので、余裕があれば試してください。

例: CLAUDE.md がある状態と、一時的にリネームして無い状態とで、同じプロンプトの結果がどう変わるか。

## デモ開始前のリセット手順（任意）

何度も試して状態が散らかったら、以下できれいに戻せます。

```bash
# タスクデータを初期状態へ
git checkout data/tasks.json

# Hooks が作るログを削除
rm -f .claude/edit-log.txt

# Claude が作った実験用ファイルがあれば削除（例）
git clean -nd        # まず削除対象を確認（-n はドライラン）
git clean -fd        # 実際に削除
```

## 困ったときの確認コマンド（Claude Code 内で）

- `/help` … スラッシュコマンド一覧
- `/agents` … 利用可能なサブエージェント一覧
- `/hooks` … 設定中のフック一覧
- `/output-style` … 出力スタイルの切替
- `/context` … コンテキストの使用量を確認
- `/memory` … 読み込まれている CLAUDE.md / rules を確認
