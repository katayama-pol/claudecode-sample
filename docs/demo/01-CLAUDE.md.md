# 01. CLAUDE.md — プロジェクトの常設メモリ

## どんな機能か

`CLAUDE.md` は、Claude Code が**セッション開始時に自動で読み込む**プロジェクトの常設メモリです。
プロジェクトの概要・コマンド・規約・作業の進め方を書いておくと、**毎回指示しなくても**守ってくれます。

- 発動タイミング: セッション開始時（自動）。常に文脈の先頭に置かれる。
- 置き場所: リポジトリ直下の [CLAUDE.md](../../CLAUDE.md)（個人用は `~/.claude/CLAUDE.md`）。

## 打つプロンプト

```
このプロジェクトはどうやって実行する？テストは？
```

### 何が起きれば成功か

- README を読みにいかなくても、`npm run task -- <コマンド>` や `npm test` を即答する。
- 「ビルド不要」「データは data/tasks.json」など CLAUDE.md に書いた事実を踏まえて答える。

## もう一歩：書いた規約が効いているか

```
新しいタスクを「資料を印刷する」というタイトルで追加して。
```

### 観察ポイント

- CLAUDE.md に「data/tasks.json を勝手に書き換えない／CLI経由で追加」と書いてあるため、
  Claude はファイルを直接編集せず **`node src/index.ts add ...` を実行**しようとする。
  （※直接編集しようとしても 05-Hooks のガードでブロックされます）

## 比較実験（ON/OFF）

効果を体感するなら、一時的に CLAUDE.md を無効化して同じ質問をしてみてください。

```bash
# OFF にする
mv CLAUDE.md CLAUDE.md.bak
```

Claude Code を再起動して「このプロジェクトはどうやって実行する？」と聞くと、
今度は**ファイルを探し回ってから**答える（または曖昧になる）はずです。確認したら戻します。

```bash
mv CLAUDE.md.bak CLAUDE.md
```

## `@` インポート：他ファイルを取り込む

CLAUDE.md は `@パス` と書くことで、別ファイルの内容を**その場に取り込め**ます
（取り込まれた先がさらに `@` で取り込むことも可能で、**最大4階層**まで再帰展開されます）。
本デモでは [CLAUDE.md](../../CLAUDE.md) が `@.claude/architecture.md` で
[アーキテクチャメモ](../../.claude/architecture.md)を取り込んでいます。

```
このプロジェクトのデータフローを説明して。
```

### 何が起きれば成功か

- architecture.md に書いた「データの読み書きは store 経由」「tasks.json が唯一の永続データ」を、
  そのファイルを明示的に開かせなくても説明する（CLAUDE.md 経由で取り込まれているため）。

> 注意: この `@` インポートは **CLAUDE.md の機能**です。`.claude/rules/` のルールファイル内では
> `@` は使えません（→ [02-Rules.md](02-Rules.md)）。混同しやすいので注意。

## メリットまとめ

- チーム全員・Claude が**同じ前提**を共有できる（オンボーディングコスト減）。
- 「うちのやり方」を毎回プロンプトに書く手間がなくなる。
- リポジトリに入るので、**共有・バージョン管理**できる。
