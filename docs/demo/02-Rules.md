# 02. Rules — 規約を部品化し、条件付きでロードする

## どんな機能か

「Rules（ルール）」は、守ってほしい規約を `.claude/rules/` ディレクトリに**独立したファイル**として
置いておく仕組みです。Claude Code が自動でロードします。CLAUDE.md の `@` インポートとは**別の仕組み**です。

ロードのされ方はフロントマターの `paths` で2通りに分かれます。

| 種類 | フロントマター | ロードされるタイミング |
|---|---|---|
| 常時ロード | `paths` **なし** | セッション開始時（CLAUDE.md と同様、常に効く） |
| 条件付きロード | `paths: [...]` **あり** | Claude がそのglobに一致するファイルを**読んだとき** |

> ⚠️ 既知の注意点: `paths` のマッチは**ファイルを読んだとき**に発動します。
> 書き込み（編集）だけでは発動しないことがあります。

このデモのルール:

- [.claude/rules/coding-style.md](../../.claude/rules/coding-style.md) … `paths: ["src/**/*.ts"]`（条件付き）
- [.claude/rules/git-style.md](../../.claude/rules/git-style.md) … `paths` なし（常時）

> 補足: Rules ファイルの中では `@` インポート構文は**使えません**（`@` は CLAUDE.md 専用）。

## デモA：常時ロードのルール（git-style）

```
このプロジェクトのコミットメッセージの規約は？
```

### 何が起きれば成功か

- `paths` なしの git-style.md が常時ロードされているため、何も読ませなくても
  「Conventional Commits 形式」「種別は feat/fix/...」と即答する。

## デモB：条件付きロードのルール（coding-style）

coding-style.md は `src/**/*.ts` を**読んだとき**に初めてロードされます。差を体感します。

### B-1. まだ src を読んでいない状態で聞く

セッションを開き直した直後（src のファイルにまだ触れていない状態）で:

```
このプロジェクトの「ユーザー向けメッセージ」の言語ルールは？根拠ファイルも教えて。
```

→ この時点では coding-style.md が**まだロードされていない**ことがあります
（一般論で答える、確証が弱い、など）。

### B-2. src を読ませてから、もう一度聞く

```
src/index.ts を読んで、そのうえで「ユーザー向けメッセージ」の言語ルールと根拠を教えて。
```

### 何が起きれば成功か

- `src/index.ts`（`src/**/*.ts` にマッチ）を読んだことで coding-style.md がロードされ、
  「メッセージはすべて日本語」「エラーは `エラー: ` で始める」と
  **coding-style.md を根拠に**答えるようになる。
- B-1 と B-2 で答えの確度・根拠の有無が変わるのが、条件付きロードの体感ポイントです。

## メリットまとめ

- 規約を**テーマ別ファイルに分割**して管理できる（再利用・差分レビューがしやすい）。
- `paths` で**関連する作業のときだけ**ルールを効かせられる → 文脈の無駄遣いを防ぐ。
- CLAUDE.md を肥大化させずに済む。
