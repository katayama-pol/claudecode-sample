# 04. Subagents — 別コンテキストの専門家に委譲する

## どんな機能か

Subagent（サブエージェント）は、特定タスク専用の**別エージェント**です。
独立したコンテキスト（会話履歴）で動くので、メインの会話を汚さずに集中して処理させられます。
使えるツールやモデルも個別に絞れます。

- 発動タイミング:
  - **明示的**: 「code-reviewer サブエージェントでレビューして」と指定。
  - **自動**: `description` に合う作業を Claude が委譲することがある。
- 置き場所: [.claude/agents/code-reviewer.md](../../.claude/agents/code-reviewer.md)。
  フロントマターで `name` / `description` / 使える `tools` / `model` を定義する。
- 確認: Claude Code 内で `/agents` を打つと一覧に `code-reviewer` が出ます。

## 事前準備：レビュー対象の変更を作る

まずレビューする差分を用意します（03-Skills で何か追加していればそれでもOK）。
無ければ、わざと規約違反を入れてもらいます:

```
src/index.ts の help() の使い方一覧に、英語で "  ping   pong" という行を1行だけ足して。
```

（メッセージは日本語、という規約にわざと反する変更です）

## 打つプロンプト

```
code-reviewer サブエージェントを使って、いまの変更をレビューして。
```

### 何が起きれば成功か

- メインの会話とは**別のエージェント**が起動し、`git diff` を見て規約照合する。
- 「英語のメッセージは規約違反（日本語にすべき）」を**指摘**して返す。
- レビュー結果だけがメイン会話に戻り、途中の探索ログでメイン文脈が膨らまない。

## なぜ別コンテキストが嬉しいか

- レビューのための大量のファイル読み込みが、**メインの会話履歴を圧迫しない**。
- 役割ごとにモデルやツール権限を変えられる（例: レビューは読み取り＋Bashのみ、書き込み不可）。
- 複数のサブエージェントを**並列**に走らせれば、調査やレビューを同時進行できる。

## メリットまとめ

- 専門タスクを**隔離して集中処理**（文脈汚染を防ぐ）。
- 役割ごとの**権限・モデルの最適化**。
- 並列化でスループット向上。

> 後片付け: 実験用に入れた英語の行は `git checkout src/index.ts` で戻せます。
