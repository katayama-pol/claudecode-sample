# 06. Output styles — 応答の振る舞い・口調を切り替える

## どんな機能か

Output style（出力スタイル）は、Claude の**システムプロンプトそのものを差し替える**機能です。
非デフォルトのスタイルを選ぶと、Claude Code 組み込みのコーディング指示（簡潔に答える・
テストで検証する等）が**置き換えられます**（＝CLAUDE.md が「知識を足す」のに対し、
Output style は「土台の振る舞いそのもの」を変える、影響範囲の大きい機能です）。

> 補足: スタイル側のフロントマターに `keep-coding-instructions: true` を付けると、
> 組み込みのコーディング指示を**維持**したまま独自スタイルを上乗せできます。
> このデモの「先輩メンター」はこの指定を付けてあるので、テスト検証などの土台は保たれます。

このデモには「**先輩メンター**」スタイルを用意しています。
（定義: [.claude/output-styles/senpai-mentor.md](../../.claude/output-styles/senpai-mentor.md)）

- 発動タイミング: `/output-style` で選択している間、ずっと。
- 選択内容はプロジェクトローカルの `.claude/settings.local.json`（`outputStyle`）に保存され、
  **セッションをまたいで永続**します（標準に戻すまで継続）。
- 組み込みスタイルは **Default / Proactive / Explanatory / Learning** の4種類。
  本デモの「先輩メンター」はカスタムスタイルです。

## 手順

### 1. まず標準スタイルで実行（比較の基準）

```
src/store.ts に、IDでタスクを1件探す findById 関数を追加して。
```

→ 簡潔に作業して終わるはずです。この応答を覚えておきます。

### 2. スタイルを切り替える

```
/output-style
```

一覧から **「先輩メンター」** を選択します。

### 3. 同じ系統の作業をもう一度

```
src/store.ts に、未完了タスクだけ返す listPending 関数を追加して。
```

### 何が起きれば成功か（標準との差）

先輩メンタースタイルでは、同じ作業でも応答が変わります:

- 作業前に「これから何をするか」を予告する。
- 実装後に「**なぜその書き方にしたか**」を一言添える。
- 専門用語に短い補足が付く。
- 最後に「**今回のポイント**」として学びがまとまる。

→ 同じタスクでも、**説明の厚みと構成**がはっきり変わるのが体感できます。

### 4. 元に戻す

```
/output-style
```

から「Default」（標準）を選び直します。

## CLAUDE.md との違い（混同しやすい）

| | CLAUDE.md | Output style |
|---|---|---|
| 変えるもの | 知っている**前提・知識** | **振る舞い・伝え方** |
| 例 | 「データは tasks.json」 | 「理由を添えて説明する」 |
| 併用 | 同時に効く | 同時に効く |

## メリットまとめ

- 用途に合わせて Claude の**働き方を切り替え**られる（教育モード／簡潔モード等）。
- チームで共有すれば、**説明の粒度を揃え**られる。
- プロジェクトの知識（CLAUDE.md）とは独立に管理できる。

> 後片付け: 追加した `findById` / `listPending` が不要なら `git checkout src/store.ts` で戻せます。
> スタイルは必ず Default に戻しておくと、次のデモがやりやすいです。
