# 07. system prompt 追記 — その場限りの方針を注入する

## どんな機能か

Claude Code を起動するときに `--append-system-prompt` オプションを付けると、
**そのセッションの間だけ**有効な追加指示をシステムプロンプトに足せます。

- CLAUDE.md … リポジトリに**常設**（全員・毎回）
- Output style … 切り替えている**間**ずっと
- **system prompt 追記 … その起動のセッション限り**（一時的・実験的）

「今日はこの方針で試したい」「この1セッションだけルールを変えたい」という
**使い捨ての方針注入**に向いています。

## 手順

いったん Claude Code を終了し、オプション付きで起動し直します。

```bash
# 例: このセッション中は、回答の冒頭に必ず合言葉を付けさせる
claude --append-system-prompt "あなたの回答は必ず『【実験セッション】』という文字列から始めること。"
```

起動したら、何でもいいので質問します。

```
このプロジェクトのテストはどう動かす？
```

### 何が起きれば成功か

- 回答が必ず **「【実験セッション】」** で始まる。
- いったん `claude` を**オプション無しで**起動し直すと、合言葉は付かなくなる
  （= その場限りだったことが分かる）。

## もう少し実務寄りの例

```bash
# レビュー観点を一時的に上書きしたいとき
claude --append-system-prompt "今回はセキュリティ観点を最優先し、入力検証の不足を必ず指摘すること。"

# 出力言語をそのセッションだけ英語にしたいとき
claude --append-system-prompt "Respond in English for this session only."
```

> 補足: `--system-prompt`（完全置換）系のオプションもありますが、通常は
> 既存の振る舞いを保ったまま足せる `--append-system-prompt` が安全で使いやすいです。
> 厳密なオプション名は `claude --help` で確認してください（バージョンで差があります）。

## 使い分けの早見表

| やりたいこと | 使うもの |
|---|---|
| 全員・毎回守らせたい | **CLAUDE.md / Rules** |
| 説明の仕方をしばらく変えたい | **Output style** |
| この1セッションだけ試したい | **system prompt 追記** |
| ツール実行を確実に制御したい | **Hooks** |
| 定型手順を再現したい | **Skills** |
| 専門処理を隔離したい | **Subagents** |

## メリットまとめ

- リポジトリの設定を汚さずに、**一時的な方針**を試せる。
- A/B 的に「この指示を足すと挙動がどう変わるか」を**実験**できる。
- スクリプトやCIから、起動ごとに方針を**動的に注入**できる。
